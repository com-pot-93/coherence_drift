The process begins when temperature values are obtained. After collecting these values, a decision is made based on whether a certain condition is met. If the condition is met, a standard notification is generated. If the condition is not met, the process skips this step. Regardless of the outcome, the process continues to the next phase.

At this point, three activities are carried out independently and at the same time:

1. **First activity:** The status of the container is retrieved. Once this information is available, the condition of the container is estimated. Based on this estimation, a decision is made about whether a change to the container is needed. If a change is needed, a request for a container change is made. If no change is needed, this step is skipped. After this, the process continues.

2. **Second activity:** The location of the client is obtained. Using this information, the distance to the client is estimated. After estimating the distance, a decision is made about whether a delivery time estimation is needed. If it is needed, the delivery time is estimated. Then, another decision is made about whether a change to the delivery route is required. If a route change is needed, a request for a route change is made. If no route change is needed, this step is skipped. If no delivery time estimation is needed, the process skips directly to the next phase. After all these steps, the process continues.

3. **Third activity:** The status of the goods is obtained. Once this is done, the condition of the goods is estimated. Based on this estimation, a decision is made about whether damages are suspected. If damages are suspected, the damages are evaluated. After evaluation, a decision is made about whether the goods need to be removed. If removal is needed, a request for removal of the goods is made. If removal is not needed, a request for a price change is made. After this, the process continues. If no damages are suspected, a request for a price change is made directly. After the request for a price change, the process continues.

Once all three activities and their related decisions and actions are completed, the process moves to the final step, where all generated messages are sent. After sending the messages, the process ends.
