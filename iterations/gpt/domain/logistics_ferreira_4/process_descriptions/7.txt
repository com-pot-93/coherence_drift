The process begins with the collection of temperature values. Once these values are gathered, a decision is made based on whether a certain condition is met. If the condition is met, a standard notification is generated. If the condition is not met, the process skips the notification step. Regardless of the outcome, the process continues to the next phase.

At this point, three activities are carried out at the same time:

1. **First activity:** The status of the container is retrieved. After obtaining this information, the condition of the container is estimated. Based on this estimation, a decision is made about whether a change to the container is needed. If a change is needed, a request for a container change is made. If no change is needed, the process continues without making a request.

2. **Second activity:** The location of the client is obtained. Using this information, the distance to the client is estimated. After estimating the distance, a decision is made about whether an estimation of delivery time is needed. If it is needed, the delivery time is estimated. Then, another decision is made about whether a change to the delivery route is required. If a route change is needed, a request for a route change is made. If no change is needed, the process continues without making a request. If no estimation of delivery time is needed, the process skips directly to the next phase.

3. **Third activity:** The status of the goods is obtained. After this, the condition of the goods is estimated. Based on this estimation, a decision is made about whether damages are suspected. If damages are suspected, the damages are evaluated. After evaluation, a decision is made about whether the goods need to be removed. If removal is needed, a request for goods removal is made. If no removal is needed, the process continues without making a request. If no damages are suspected, the process skips the evaluation and removal steps and proceeds to request a price change.

Once all three activities are completed, the process continues by sending all generated messages. After this, the process ends.
