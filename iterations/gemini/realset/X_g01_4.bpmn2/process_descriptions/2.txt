The process starts. It then enters a loop where a condition is checked. If the condition is true, the process loops back to check the condition again. If the condition is false, the process proceeds to the next step, which is to select a course. After selecting a course, the process enters another loop where a condition is checked. If the condition is true, the process loops back to check the condition again. If the condition is false, the process continues to view dates and select a date. Next, the process splits based on whether an account is available. If an account is available, the process proceeds along that path. If an account is not available, the process splits again based on whether the user is registered. If the user is registered, the process proceeds along that path. If the user is not registered, an activation request is sent, and the process waits for a response before proceeding. After waiting for a response or if the user was already registered, the process merges back together. The process merges again with the path where an account was available. The process then requires payment information. Following this, the process splits into two parallel activities: tweeting friends and receiving a ticket. Both activities occur simultaneously. Once both activities are completed, the process merges back together and ends.
