The process starts, and then proceeds to a decision point. If "Condition 1" is met, a task labeled "Loop 1" is performed, after which the process returns to the same decision point. Otherwise, the process moves to another decision point. If "Condition 2" is met, a task labeled "Loop 2" is performed, and the process returns to this decision point. Otherwise, a "Select a course" task is performed, followed by another decision point. If "Condition 3" is met, a task labeled "Loop 3" is performed, and the process returns to this decision point. Otherwise, the process continues with a "View dates" task, followed by a "Select a date" task, and then another decision point. From here, the process splits into two possible paths based on whether a user is registered or not. If the user is registered, the process continues. If the user is not registered, a "Send activation request" task is performed, and then the process continues. Alternatively, if neither of the above conditions is met, a "Provide payment information" task is performed, and then the process continues. The process then proceeds to a parallel gateway, which initiates two tasks in parallel: "Tweet friends" and "Receive ticket". After both tasks are completed, the process ends.
