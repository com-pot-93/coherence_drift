The process starts. It then enters a loop where it stays as long as a certain condition is true. Once that condition is no longer true, the process moves to another loop that also continues as long as a different condition is true. When this second condition is no longer true, a course is selected. Following this, the process enters a third loop that continues as long as yet another condition is true. When this third condition is no longer true, dates are viewed and a date is selected. Next, there is a split based on whether an account is available. If an account is available, the process proceeds. If not, there is another split based on whether the user is registered. If the user is registered, the process proceeds. If the user is not registered, an activation request is sent. After sending the activation request or if the user was already registered, the process continues. After the account availability split, payment information is provided. The process then splits into two parallel activities: tweeting friends and receiving a ticket. Both activities occur simultaneously. Once both activities are completed, the process ends.
