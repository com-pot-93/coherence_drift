The process starts with sending out invitations. After the invitations are sent, the game preferences are requested. Once the game preferences are known, the games are chosen. Following this, the process splits into two parallel paths.

In the first path, a decision is made. If the decision is true, a LAN party date is proposed, and a request to send the date is made. Subsequently, another decision is made. If the date needs to be changed, the process loops back to propose a new LAN party date. If an issue arises during proposing a LAN party date, the process escalates and returns to the decision point. This loop continues until the date is accepted. The process then proceeds to the merging gateway.

In the second parallel path, a decision is made. If the decision is true, guests are asked about the games. Following this, another decision is made. If the games need to be changed, the process loops back to ask the guests about the games again. If an issue arises during asking guests about games, the process escalates and returns to the decision point. This loop continues until the games are accepted. The process then proceeds to the merging gateway.

After both parallel paths are completed, the process continues with downloading the games. Following this, beer preferences are requested and received. A decision is then made based on whether any beer is missing. If beer is missing, the missing beer is bought. Otherwise, if all the beer is present or after the missing beer is bought, the process continues with preparing for the LAN party, and the process ends.
