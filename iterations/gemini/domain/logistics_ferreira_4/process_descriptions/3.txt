The process starts with retrieving temperature values. Based on these values, the process either generates a standard notification or gathers more information.

If a standard notification is generated, the process proceeds to send generated messages and then ends. Alternatively, after generating a standard notification, the process can escalate and then proceed to send generated messages before ending.

If more information is needed, the process retrieves container status, client location, and goods status in parallel.

For the container status, the process estimates the container status and then decides whether to generate a container change request or proceed without it. If a container change request is generated, the process can either proceed to send generated messages and then ends, or it can loop back to retrieve the container status again. If the process proceeds without generating a container change request, it can either proceed to send generated messages and then ends, or it can loop back to retrieve the container status again. The process can also escalate and then loop back to retrieve the container status again.

For the client location, the process estimates the distance and then decides whether to estimate the delivery time or proceed without estimating. If the delivery time is estimated, the process generates a route change request. If the process proceeds without estimating the delivery time, the process can either proceed to send generated messages and then ends, or it can loop back to retrieve the client location again. If a route change request is generated, the process can either proceed to send generated messages and then ends, or it can loop back to retrieve the client location again. The process can also escalate and then loop back to retrieve the client location again.

For the goods status, the process estimates the goods status and then decides whether to evaluate damages or proceed without evaluating. If damages are evaluated, the process decides whether to generate a removal request or a price change request. If the process proceeds without evaluating damages, the process can either proceed to send generated messages and then ends, or it can loop back to retrieve the goods status again. If a removal request or a price change request is generated, the process can either proceed to send generated messages and then ends, or it can loop back to retrieve the goods status again. The process can also escalate and then loop back to retrieve the goods status again.

After retrieving container status, client location, and goods status, the process proceeds to send generated messages and then ends. Alternatively, after retrieving container status, client location, and goods status, the process can escalate and then proceed to send generated messages before ending.
