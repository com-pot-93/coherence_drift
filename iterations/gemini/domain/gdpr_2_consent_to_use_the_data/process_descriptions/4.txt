The process starts. Then, the process splits into multiple parallel paths.

In the first path, the "Add Profiling Information" task is performed. After that, this path waits for the completion of all other parallel paths before proceeding.

In the second path, a decision is made. If the answer is "Yes", the "Retrieve Period" task is performed. If the answer is "No", the "Retrieve Criteria" task is performed. After either of these tasks, this path waits for the completion of all other parallel paths before proceeding.

In the third path, the "Add Right to Rectify Info" task is performed, and then this path waits for the completion of all other parallel paths before proceeding.

In the fourth path, the "Add Right to Access Info" task is performed, and then this path waits for the completion of all other parallel paths before proceeding.

In the fifth path, the "Add Right to Erase Info" task is performed, and then this path waits for the completion of all other parallel paths before proceeding.

In the sixth path, the "Add Right to Data Portability Info" task is performed, and then this path waits for the completion of all other parallel paths before proceeding.

In the seventh path, the "Add Right to Withdraw Info" task is performed, and then this path waits for the completion of all other parallel paths before proceeding.

In the eighth path, the "Add Right to Lodge a Complaint Info" task is performed, and then this path waits for the completion of all other parallel paths before proceeding.

In the ninth path, a decision is made. If the answer is "Yes", this path proceeds directly to a merging point. If the answer is "No", the "Retrieve Source" task is performed before reaching the merging point. After either of these options, this path waits for the completion of all other parallel paths before proceeding.

In the tenth path, a decision is made. If the answer is "Yes", this path proceeds directly to a merging point. If the answer is "No", the "Add Info About International Transfer" task is performed before reaching the merging point. After either of these options, this path waits for the completion of all other parallel paths before proceeding.

In the eleventh path, a decision is made. If the answer is "Yes", this path proceeds directly to a merging point. If the answer is "No", the "Retrieve Recipients" task is performed before reaching the merging point. After either of these options, this path waits for the completion of all other parallel paths before proceeding.

In the twelfth path, the "Retrieve Processing Legal Basis" task is performed, and then this path waits for the completion of all other parallel paths before proceeding.

In the thirteenth path, the "Retrieve Processing Purposes" task is performed, and then this path waits for the completion of all other parallel paths before proceeding.

Once all the parallel paths have completed, the process splits into two parallel paths again.

In the first of these new paths, the "Retrieve Identity and Contact Details" task is performed. After that, this path waits for the completion of the other parallel path before proceeding.

In the second of these new paths, the "Ask for DPO Details" task is performed, followed by the "Receive DPO Details" task. After that, this path waits for the completion of the other parallel path before proceeding.

Once both of these parallel paths have completed, the "Collect Consent Information" task is performed, followed by the "Ask for Consent" task. A decision is then made based on the response to the consent request.

If "Consent Received", the "Receive Consent" task is performed. If "Consent Denied", the "Receive Negation" task is performed.

Finally, after either the "Receive Consent" or "Receive Negation" task, the process ends.
