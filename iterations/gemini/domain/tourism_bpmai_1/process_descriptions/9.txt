The process starts when a customer request is received. The request is then checked. If no registration is required, the process proceeds directly to the "Continue Process" step. Otherwise, the process branches into either manual or automatic registration. In the case of manual registration, a manual registration task is performed. Alternatively, for automatic registration, an automatic registration task is performed. After either manual or automatic registration, the process merges and proceeds to the "Continue Process" step.

From "Continue Process", the process splits again. One path skips the attraction provider search, while the other path searches for an attraction provider. If the search is performed, a condition is checked. If the condition is met, the process merges to a later step. If the condition is not met, an attraction is booked, and the booking status is checked. If the booking is confirmed, a confirmation is sent. If the booking is rejected, a rejection notification is sent. After sending either the confirmation or rejection notification, the process merges and rejoins the path where the condition was met.

The two paths then merge, proceeding to the "Continue" task. Finally, a notification is sent, and the process ends.
