The process starts when a change request is received. This request is then reviewed. Depending on the outcome of the review, the process either proceeds with a rejection task if the request is rejected, or a confirmation task if the request is confirmed. After either the rejection or confirmation task is completed, a change is performed. Following the change, the process either proceeds with reporting the implementation if the implementation is reported, or notifying about a failure if a failure notification is received. Finally, after either reporting the implementation or notifying about a failure, the process ends.
