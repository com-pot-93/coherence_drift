The process starts when a change request is received. This request is then reviewed. Depending on the outcome of the review, the request is either rejected or confirmed. If rejected, a rejection task is performed. If confirmed, a confirmation task is performed. After either the rejection or confirmation task, the change is performed. Following the change, the process branches based on whether the implementation was reported or a failure notification was received. If the implementation was reported, a task to report the implementation is performed. If a failure notification was received, a notification about the failure is sent. After either reporting the implementation or notifying about the failure, the process ends.
