The process starts when a change request is received. This request is then reviewed. Depending on the outcome of the review, the change request is either rejected or confirmed. If rejected, a rejection task is performed. If confirmed, a confirmation task is performed. After either rejection or confirmation, the change is performed. Following the change, it is determined whether the implementation is reported or a failure notification is sent. In the case of a reported implementation, a task to report the implementation is performed. In the case of a failure notification, a task to notify about the failure is performed. Finally, after either reporting the implementation or notifying about the failure, the process ends.
