The process starts when a change request is received. This request is then reviewed. Depending on the outcome of the review, the process follows one of two paths. If the request is rejected, a rejection task is performed. If the request is confirmed, a confirmation task is performed instead. After either the rejection or confirmation task is completed, a change is performed. Following the change, the process again splits into two possible paths. If the implementation is reported, a task to report the implementation is performed. Alternatively, if a failure notification is received, a notification about the failure is sent. After either reporting the implementation or notifying about the failure, the process ends.
